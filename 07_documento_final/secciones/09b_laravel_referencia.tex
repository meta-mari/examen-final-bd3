\section{Ejecución de la aplicación Laravel del proyecto}

Además del diseño de base de datos desarrollado para el examen, se ejecutó la aplicación web Laravel del sistema odontológico LALYSDENT. Esta aplicación corresponde al repositorio desarrollado por el equipo del proyecto, disponible en \url{https://github.com/AlanDevPro/sistema-odontologico} \parencite{sistemaodontologicogithub}.

La ejecución de la aplicación no cambia el objetivo central del informe. El producto principal sigue siendo la base de datos Oracle 19c diseñada, documentada y validada en el schema \texttt{FARMACIAS\_BD3}. La aplicación Laravel se incluye para evidenciar el funcionamiento del sistema web asociado al mismo dominio odontológico y para complementar la comparación entre la lógica de aplicación y la lógica implementada directamente en base de datos.

\subsection{Relación con la tesis base y con el sistema desarrollado}

La tesis base documenta un sistema de información web para la administración de procesos de registro, atención, inventario y finanzas del consultorio odontológico LALYSDENT. El sitio web de la tesis, disponible en \url{https://calvocobos.github.io/}, se usó como apoyo documental del dominio \parencite{calvocobos2025web}.

El repositorio Laravel del equipo representa la implementación web funcional del sistema y forma parte del mismo proyecto. Su ejecución permitió comprobar la autenticación, las migraciones, los seeders, el panel principal y módulos funcionales como pacientes, citas, pagos, compras e inventario.

\subsection{Separación entre schemas Oracle}

Para mantener separadas las responsabilidades, se trabajó con dos schemas distintos en Oracle:

\begin{table}[H]
\caption{Separación de schemas Oracle}
\centering
\small
\begin{tabularx}{\textwidth}{L{0.28\textwidth}Y}
\toprule
\textbf{Schema} & \textbf{Uso} \\
\midrule
\texttt{FARMACIAS\_BD3} & Schema diseñado, documentado y validado como producto principal del examen final. Contiene la base construida manualmente mediante SQL y PL/SQL. \\
\texttt{LALYSDENT\_APP} & Schema usado para ejecutar la aplicación Laravel del proyecto mediante sus propias migraciones y seeders, sin modificar el diseño del examen. \\
\bottomrule
\end{tabularx}
\end{table}

Esta separación es importante porque la aplicación Laravel ya posee su propia estructura de base de datos y su propia lógica de aplicación. Por ello, no fue necesario conectarla al schema \texttt{FARMACIAS\_BD3}. El schema \texttt{FARMACIAS\_BD3} se conserva como diseño independiente de base de datos, mientras que \texttt{LALYSDENT\_APP} se usa para ejecutar el sistema web.

\subsection{Resultado}

La ejecución de la aplicación Laravel demuestra que el sistema web del proyecto funciona correctamente en el dominio odontológico. Sin embargo, el objetivo principal de este informe sigue siendo el diseño, transformación, implementación y validación de la base de datos Oracle 19c desarrollada para el examen. La aplicación Laravel se documenta como implementación web propia del equipo y como evidencia funcional complementaria, manteniendo separado el diseño independiente realizado en \texttt{FARMACIAS\_BD3}.
